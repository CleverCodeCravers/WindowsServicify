# User Story: CLAUDE.md erstellen (R00017)

## Narrativ

Als AI-Assistent (z.B. Claude, Copilot), der mit dem WindowsServicify-Repository arbeitet,
moechte ich eine CLAUDE.md im Repository-Root vorfinden,
damit ich sofort den Projektkontext verstehe -- Build-Befehle, Struktur, Konventionen --
und effizient bei Implementierungen, Reviews und Fehlerbehebungen helfen kann,
ohne erst die gesamte Codebasis explorieren zu muessen.

## Szenarien

### Szenario 1: Build-Anweisungen sind dokumentiert

**Given** die Datei `CLAUDE.md` existiert im Repository-Root
**When** ein AI-Assistent den Inhalt liest
**Then** findet er Anweisungen zum Bauen (`dotnet build`) und Testen (`dotnet test`) des Projekts
**And** den Pfad zur Solution-Datei

### Szenario 2: Projektstruktur ist erklaert

**Given** die Datei `CLAUDE.md` existiert im Repository-Root
**When** ein AI-Assistent den Inhalt liest
**Then** versteht er die Rolle jedes Projekts:
  - ConsoleApp = Entry Point / Host
  - Domain = Business Logic
  - Domain.Tests = Unit-Tests

### Szenario 3: Plattform-Einschraenkung ist dokumentiert

**Given** die Datei `CLAUDE.md` existiert im Repository-Root
**When** ein AI-Assistent den Inhalt liest
**Then** erkennt er, dass das Projekt ausschliesslich unter Windows laeuft (kein Cross-Platform)

### Szenario 4: Konventionen sind beschrieben

**Given** die Datei `CLAUDE.md` existiert im Repository-Root
**When** ein AI-Assistent den Inhalt liest
**Then** kennt er die Projekt-Konventionen:
  - Nullable Reference Types aktiviert
  - Record-Types fuer DTOs
  - NUnit 4.x als Test-Framework

### Szenario 5: Anforderungen-Verzeichnis ist referenziert

**Given** die Datei `CLAUDE.md` existiert im Repository-Root
**When** ein AI-Assistent den Inhalt liest
**Then** weiss er, dass Anforderungen im Verzeichnis `Anforderungen/` gepflegt werden

## Mehrwert

Ohne CLAUDE.md muss jeder AI-Assistent bei jeder Interaktion die Projektstruktur neu erkunden --
Solution-Pfad suchen, Framework-Version herausfinden, Test-Framework identifizieren.
Mit CLAUDE.md steht dieser Kontext sofort bereit, was die Qualitaet und Geschwindigkeit
der AI-gestuetzten Arbeit deutlich verbessert.
