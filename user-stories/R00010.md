# R00010: Upgrade auf .NET 8.0

## User Story

**Als** Entwickler und Betreiber von WindowsServicify
**moechte ich** dass das Projekt auf .NET 8.0 (LTS) aktualisiert wird,
**damit** ich von Long-Term-Support, Performance-Verbesserungen und aktuellen Security-Patches profitiere und das Projekt nicht auf einem End-of-Life-Framework (.NET 7.0) laeuft.

## Mehrwert

.NET 7.0 hat am 14. Mai 2024 sein Support-Ende erreicht. Ein Upgrade auf .NET 8.0 (LTS, Support bis November 2026) stellt sicher, dass:
- Sicherheitsupdates weiterhin verfuegbar sind
- Die CI/CD-Pipeline mit aktuellen SDK-Versionen arbeitet
- NuGet-Pakete in aktuellen, kompatiblen Versionen genutzt werden
- Das Projekt fuer zukuenftige Weiterentwicklung auf einer stabilen Basis steht

## Akzeptanzkriterien und Szenarien

### Szenario 1: Target Framework ist .NET 8.0

**Given** die drei Projektdateien (ConsoleApp, Domain, Domain.Tests)
**When** ich die TargetFramework-Eigenschaft pruefe
**Then** steht in allen drei csproj-Dateien `net8.0`

### Szenario 2: NuGet-Pakete sind aktualisiert

**Given** das Domain-Projekt mit Abhaengigkeiten auf Microsoft.Extensions.Hosting.WindowsServices und CommandLineArgumentsParser
**When** ich die PackageReference-Versionen pruefe
**Then** ist Microsoft.Extensions.Hosting.WindowsServices auf Version 8.0.x aktualisiert
**And** CommandLineArgumentsParser bleibt auf 1.0.7 (keine neuere Version verfuegbar, kompatibel mit .NET 8.0)

### Szenario 3: Test-Pakete sind aktualisiert

**Given** das Domain.Tests-Projekt mit NUnit und Microsoft.NET.Test.Sdk
**When** ich die PackageReference-Versionen pruefe
**Then** sind alle Test-Pakete auf aktuelle, .NET-8.0-kompatible Versionen aktualisiert

### Szenario 4: CI-Workflow nutzt .NET 8.0 SDK

**Given** die GitHub Actions Workflow-Datei (.github/workflows/build.yml)
**When** ich die DOTNET_VERSION Umgebungsvariable pruefe
**Then** steht dort `8.0.x`
**And** die actions/checkout und actions/setup-dotnet verwenden aktuelle Versionen

### Szenario 5: Build ist erfolgreich (mit bekannter Einschraenkung)

**Given** alle Aenderungen sind durchgefuehrt
**When** ich `dotnet build` ausfuehre
**Then** kompiliert das Projekt erfolgreich
**Or** es gibt einen erwarteten Fehler wegen CommandLineArgumentsParser-Kompatibilitaet, der separat behandelt wird
