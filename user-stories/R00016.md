# User Story: R00016 - CI-Workflow modernisieren

## Wer nutzt es?

Als **Entwickler des WindowsServicify-Projekts** moechte ich einen zuverlaessigen und aktuellen CI-Workflow haben, damit ich bei jedem Push automatisch sichergestellt bekomme, dass Build und Tests erfolgreich durchlaufen, und damit Abhaengigkeiten automatisch auf Aktualitaet geprueft werden.

## Mehrwert aus Nutzersicht

Ein moderner CI-Workflow ist das Rueckgrat der Softwarequalitaet. Ohne aktiven Test-Schritt in der Pipeline koennen fehlerhafte Aenderungen unbemerkt in den main-Branch gelangen. Ohne Dependabot bleiben veraltete oder unsichere Abhaengigkeiten unentdeckt. Die Modernisierung des Workflows schuetzt die Codequalitaet, reduziert manuellen Pruefaufwand und haelt das Projekt sicherheitstechnisch aktuell.

## Akzeptanzkriterien

### Szenario 1: Test-Schritt ist im CI aktiv

**Given** der CI-Workflow wird bei einem Push ausgefuehrt
**When** der Build-Schritt erfolgreich abgeschlossen wurde
**Then** wird automatisch `dotnet test` ausgefuehrt und das Ergebnis bestimmt den Pipeline-Status

### Szenario 2: Dependabot fuer NuGet konfiguriert

**Given** die Datei `.github/dependabot.yml` existiert
**When** Dependabot das Repository analysiert
**Then** werden NuGet-Paketabhaengigkeiten woechentlich auf Aktualisierungen geprueft und bei Bedarf automatisch Pull Requests erstellt

### Szenario 3: Dependabot fuer GitHub Actions konfiguriert

**Given** die Datei `.github/dependabot.yml` existiert
**When** Dependabot das Repository analysiert
**Then** werden GitHub-Actions-Versionen woechentlich auf Aktualisierungen geprueft und bei Bedarf automatisch Pull Requests erstellt

### Szenario 4: Actions sind auf aktuellem Stand

**Given** der CI-Workflow `build.yml` wird inspiziert
**When** die Action-Versionen geprueft werden
**Then** verwenden `actions/checkout` und `actions/setup-dotnet` jeweils Version `@v4`

### Szenario 5: .NET SDK-Version ist aktuell

**Given** der CI-Workflow `build.yml` wird inspiziert
**When** die konfigurierte .NET-SDK-Version geprueft wird
**Then** ist diese auf `8.0.x` (LTS) gesetzt
