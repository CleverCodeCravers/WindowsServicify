# User Story: R00011 - CommandLineArgumentsParser durch eigenen Parser ersetzen

## Wer nutzt es?

Als **Entwickler**, der das Projekt klonen und bauen moechte,
moechte ich, dass das Projekt **ohne privaten NuGet-Feed baubar** ist,
damit ich ohne spezielle Zugangsdaten am Projekt arbeiten kann.

Als **CI/CD-Pipeline**,
moechte ich keine Secrets fuer private NuGet-Feeds benoetigen,
damit der Build-Prozess einfacher und zuverlaessiger wird.

## Mehrwert

- Das Projekt wird unabhaengig von privater Infrastruktur
- Neue Entwickler koennen sofort loslegen ohne Feed-Zugangsdaten
- Die CI-Pipeline wird vereinfacht (kein NUGET_PAT Secret mehr noetig)
- Weniger externe Abhaengigkeiten = weniger Risiko bei Breaking Changes

## Akzeptanzkriterien als Given/When/Then

### Szenario 1: Projekt ist ohne privaten NuGet-Feed baubar
- **Given** das Projekt wurde frisch geklont
- **When** `dotnet build` ohne spezielle NuGet-Konfiguration ausgefuehrt wird
- **Then** baut das Projekt erfolgreich ohne Fehler

### Szenario 2: --configure wird korrekt erkannt
- **Given** die Anwendung wird mit `--configure` aufgerufen
- **When** der Parser die Argumente verarbeitet
- **Then** ist `Configure = true` und alle anderen Optionen `false`

### Szenario 3: --install wird korrekt erkannt
- **Given** die Anwendung wird mit `--install` aufgerufen
- **When** der Parser die Argumente verarbeitet
- **Then** ist `Install = true` und alle anderen Optionen `false`

### Szenario 4: --uninstall wird korrekt erkannt
- **Given** die Anwendung wird mit `--uninstall` aufgerufen
- **When** der Parser die Argumente verarbeitet
- **Then** ist `Uninstall = true` und alle anderen Optionen `false`

### Szenario 5: --testrun wird korrekt erkannt
- **Given** die Anwendung wird mit `--testrun` aufgerufen
- **When** der Parser die Argumente verarbeitet
- **Then** ist `Testrun = true` und alle anderen Optionen `false`

### Szenario 6: --legacy wird korrekt erkannt (Kombination mit --install)
- **Given** die Anwendung wird mit `--install --legacy` aufgerufen
- **When** der Parser die Argumente verarbeitet
- **Then** ist `Install = true` und `Legacy = true`

### Szenario 7: --help wird korrekt erkannt
- **Given** die Anwendung wird mit `--help` aufgerufen
- **When** der Parser die Argumente verarbeitet
- **Then** ist `Help = true` und alle anderen Optionen `false`

### Szenario 8: Unbekannte Argumente fuehren zu Fehler
- **Given** die Anwendung wird mit `--unknown` aufgerufen
- **When** der Parser die Argumente verarbeitet
- **Then** gibt der Parser einen Fehler zurueck (IsSuccess = false)

### Szenario 9: Leere Argumente fuehren zu Fehler
- **Given** die Anwendung wird ohne Argumente aufgerufen
- **When** der Parser die Argumente verarbeitet
- **Then** gibt der Parser einen Fehler zurueck (IsSuccess = false)

### Szenario 10: Hilfe-Ausgabe listet alle Kommandos
- **Given** die Anwendung wird mit `--help` aufgerufen
- **When** die Hilfe angezeigt wird
- **Then** werden alle verfuegbaren Kommandos mit Beschreibungen aufgelistet

### Szenario 11: Privater NuGet-Feed aus CI entfernt
- **Given** die build.yml Datei
- **When** der CI-Workflow geprueft wird
- **Then** existiert kein Schritt zum Hinzufuegen eines privaten NuGet-Feeds
