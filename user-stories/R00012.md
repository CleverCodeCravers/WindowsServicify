# R00012: Unit-Tests und Integration-Tests schreiben

## Wer nutzt es?

Als **Entwickler** moechte ich eine umfassende Testabdeckung fuer die Domain-Logik,
damit ich Refactorings und Erweiterungen mit Vertrauen durchfuehren kann.

## Mehrwert

Fehlende Tests fuer `ServiceConfigurationFileHandler` und `WindowsServiceInstallHelperFactory`
bedeuten, dass Aenderungen an der Serialisierung oder der Factory-Logik unbemerkt Fehler
einfuehren koennen. Round-Trip-Tests fuer die Konfiguration stellen sicher, dass gespeicherte
Konfigurationen identisch wieder geladen werden. Factory-Tests garantieren, dass die richtige
Implementierung basierend auf dem Legacy-Flag zurueckgegeben wird.

## Szenarien

### ServiceConfigurationFileHandler — Round-Trip-Serialisierung

**Szenario 1: Speichern und Laden einer gültigen Konfiguration**
- **Given** eine gueltige ServiceConfiguration mit allen Feldern gefuellt
- **When** die Konfiguration mit Save() gespeichert und mit Load() wieder geladen wird
- **Then** sind alle Felder der geladenen Konfiguration identisch mit dem Original

**Szenario 2: Speichern und Laden mit leeren optionalen Feldern**
- **Given** eine ServiceConfiguration mit leerem Arguments- und WorkingDirectory-Feld
- **When** die Konfiguration gespeichert und wieder geladen wird
- **Then** sind die leeren Felder korrekt als leere Strings erhalten

**Szenario 3: Laden einer nicht existierenden Datei**
- **Given** ein Dateipfad der nicht existiert
- **When** Load() aufgerufen wird
- **Then** wird eine FileNotFoundException geworfen

**Szenario 4: Laden einer Datei mit ungueltigem JSON**
- **Given** eine Datei mit ungueltigem JSON-Inhalt
- **When** Load() aufgerufen wird
- **Then** wird eine JsonException geworfen

**Szenario 5: Laden einer Datei mit ungueltigem ServiceName**
- **Given** eine Datei mit einer Konfiguration deren ServiceName Sonderzeichen enthaelt
- **When** Load() aufgerufen wird
- **Then** wird eine ServiceConfigurationValidationException geworfen

### WindowsServiceInstallHelperFactory — Legacy vs. PowerShell

**Szenario 6: Factory erstellt LegacyWindowsServiceInstallHelper**
- **Given** der Parameter legacy ist true
- **When** Create() aufgerufen wird
- **Then** wird eine Instanz von LegacyWindowsServiceInstallHelper zurueckgegeben

**Szenario 7: Factory erstellt PowerShellWindowsServiceInstallHelper**
- **Given** der Parameter legacy ist false
- **When** Create() aufgerufen wird
- **Then** wird eine Instanz von PowerShellWindowsServiceInstallHelper zurueckgegeben

**Szenario 8: Factory gibt immer IWindowsServiceInstallHelper zurueck**
- **Given** ein beliebiger Wert fuer legacy
- **When** Create() aufgerufen wird
- **Then** implementiert das Ergebnis das Interface IWindowsServiceInstallHelper
